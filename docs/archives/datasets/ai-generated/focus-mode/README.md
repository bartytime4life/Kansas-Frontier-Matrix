## ğŸ§  Story Node & Focus Mode Integration

This sub-layer is tightly bound to **Story Node v3** and Focus Mode itself:

- **Story Nodes**  
  - Reference archived Focus Mode corpora via `target` fields, such as:
    - `urn:kfm:corpus:focus:v11.2.4:hydrology-insight`.  
  - Use archival references to ensure time-consistent narratives when re-rendered.

- **Focus Mode**  
  - May optionally **refine** or **resummarize** from the archived corpora, instead of recomputing from raw docs, when governance allows.  
  - Uses archives as **training/evaluation corpora** for future Focus models (subject to CARE C1).

Constraints:

- Story Nodes **must not** modify archive corpora; they can only reference and describe them.  
- For Indigenous or culturally sensitive materials:
  - Story Node text must adhere to sovereignty policies.  
  - Where needed, Story Nodes may refer to higher-level summaries rather than raw or fine-grained Focus outputs.

## ğŸŒ STAC, DCAT & PROV Alignment

**STAC:**

- Collections such as:
  - `kfm-archives-ai-focus-mode-core`.

  - `properties.kfm:model_id` & `kfm:model_version`.

  - `properties.kfm:care_tag`, `kfm:governance_ledger_ref`.  
  - Links to corpora assets (JSONL, parquet, PMTiles) with checksums.

**DCAT:**

- Focus corpora appear as `dcat:Dataset` entries with:
  - Clear titles and descriptions.  
  - `dcat:distribution` entries for:
    - Narrative corpora.  
    - Span tables.  
    - Topic or cluster assignments.  
    - Provenance documentation.

**PROV-O:**

- Focus archive creation = `prov:Activity`.  
- Inputs:
  - Source datasets (`prov:used`).  
  - Model and configuration entities.  
  - Any selection filters or prompts if relevant.  
- Output:
  - Corpus entity (`prov:Entity`) with `prov:wasGeneratedBy`.  
  - Revision/supersession relationships across versions (`prov:wasRevisionOf`).

## âš– FAIR+CARE & Governance

Because Focus Mode narratives can **rephrase and refocus** sensitive source content, governance is strict:

- **Findable (F1)**  
  - Archives registered with stable IDs, indexed by STAC/DCAT, and logged in governance ledgers.

- **Accessible (A1)**  
  - Access rules follow underlying data:
    - If the underlying dataset is restricted/sensitive, Focus corpora usually are too.  
    - Public corpora are only created where governance allows.

- **Interoperable (I1)**  
  - Uses open formats and standard vocabularies (STAC/DCAT/PROV, JSON, parquet).

- **Reusable (R1)**  
  - Reuse requires:
    - Understanding constraints documented here.  
    - Respecting CARE C1 decisions and licenses.

**CARE C1 details:**

- Indigenous and community stakeholders must be able to:
  - Request restrictions or additional context on Focus corpora.  
  - Review how AI narratives represent or intersect with their histories and lands.  
- Some corpora may exist only as:
  - Restricted assets, or  
  - Aggregated/obfuscated high-level summaries.
