## üìò Overview

This directory contains **automated STAC and DCAT validation reports** generated by the `.github/workflows/stac-validate.yml` workflow.  
Each report confirms the structural integrity, metadata completeness, and FAIR alignment of KFM‚Äôs geospatial datasets.

The process verifies:
- **STAC 1.0.0 compliance** (via `pystac` and `stac-validator`)
- **DCAT 3.0 compatibility** for cross-platform discoverability
- **FAIR+CARE interoperability**, ensuring data is open, ethical, and reusable

## üßæ Report Descriptions

### 1. üß† `pystac_results.json`
Detailed record of dataset validations performed using **PySTAC** (Python STAC library).

| Field | Description |
|--------|-------------|
| `path` | File path of the validated STAC Item or Collection. |
| `ok` | Boolean indicating if the JSON passed validation. |
| `error` | Error message (if any) returned by `pystac.validate()`. |

**Example:**
```json
{
  "path": "data/stac/items/usgs_topo_1894.json",
  "ok": true,
  "error": ""
}
```

---

### 2. üîç `stac_validator_results.ndjson`
Raw NDJSON log from the official **STAC Validator CLI**, including schema references and validation context.

| Field | Description |
|--------|-------------|
| `asset` | Dataset ID or filename. |
| `valid` | Boolean result of validation. |
| `version` | STAC specification version used. |
| `errors` | List of schema or link validation errors (if any). |

**Example Line:**
```json
{"asset": "data/stac/items/ks_elevation_1950.json", "valid": true, "version": "1.0.0", "errors": []}
```

---

### 3. üìä `_summary.json`
Aggregated report combining both validation sources for high-level review and dashboard use.

| Field | Description |
|--------|-------------|
| `pystac` | Object summarizing results from PySTAC. |
| `stac_validator` | Object summarizing results from STAC Validator. |
| `timestamp` | UTC timestamp when summary was generated. |

**Example:**
```json
{
  "pystac": { "files": 52, "failed": 0 },
  "stac_validator": { "passed": 52, "failed": 0 },
  "timestamp": "2025-11-05T18:00:00Z"
}
```

## üßÆ Validation Workflow Integration

**Triggered By:**  
- Push or pull request modifying `data/stac/**`
- Manual `workflow_dispatch` runs
- Nightly scheduled checks (optional for governance audits)

**Workflow Reference:** `.github/workflows/stac-validate.yml`

**Steps Summary:**
1. Discover all `.json` files in `data/stac/`.
2. Validate each file via PySTAC.
3. Cross-check results with `stac-validator`.
4. Generate `_summary.json` and upload reports as artifacts.
5. Append run metadata to `reports/audit/github-workflows-ledger.json`.
