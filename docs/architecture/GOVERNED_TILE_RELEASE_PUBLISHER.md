## Purpose
A deterministic, no-network publishing slice that validates an approved map-layer candidate and emits governed artifacts: TileJSON, ReleaseManifest, catalog record, receipt references, and public registry entry.

## Artifact separation model
- `spec_hash` remains deterministic identity.
- PMTiles/MVT/TileJSON are render artifacts only.
- Manifest, catalog, receipts, proofs, and registry remain separate objects.
- VERIFY-before-render consumes governed release references, not raw render sources.

## Deterministic hashing model
- Canonical JSON serialization with lexicographic key ordering.
- SHA-256 over canonical JSON (or fixture bytes for pmtiles metadata fixture hash).
- `published_at` is excluded from deterministic manifest hash inputs.

## Receipt requirements
Required receipt types are declared in candidate input (`receipts.required_types`). Missing required receipts returns `NEEDS_RECEIPT`. Receipt identity mismatches (`layer_id` or `spec_hash`) are `BLOCKED`.

## Public-safe path policy
Publisher blocks RAW/WORK/QUARANTINE/canonical-private/private/restricted/draft/staging/unreleased markers in source paths and disallows non-public policy/sensitivity.

## Relationship to VERIFY-before-render
This publisher is upstream from client/API VERIFY gates. It produces references those gates use to verify governed release state before render.

## Failure outcomes
- `PUBLISHABLE`: output objects generated.
- `BLOCKED`: governance policy failure.
- `NEEDS_RECEIPT`: required receipt references missing.
- `ERROR`: unexpected runtime failure.
