# Authority ladder

KFM separates authorities so that proposals do not silently become canon.

1. **Doctrine** — `docs/doctrine/`
2. **Architecture** — `docs/architecture/` and ADRs
3. **Contracts** — `contracts/` (semantic meaning)
4. **Schemas** — `schemas/` (machine-checkable shape)
5. **Policy** — `policy/` (admissibility, sensitivity, release)
6. **Tests/Fixtures** — `tests/`, `fixtures/` (provable enforcement)
7. **Implementation** — `apps/`, `packages/`, `pipelines/`, `connectors/`, `tools/`
8. **Lifecycle data** — `data/`
9. **Release decisions** — `release/`
