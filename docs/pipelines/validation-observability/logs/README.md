## ğŸ§ª Log Validation Rules

All logs MUST:

- Validate against their corresponding schema  
- Use JSON only (no plain-text logs)  
- Contain no PII or secrets  
- Use correlation IDs consistently  
- Include provenance (`prov:wasGeneratedBy`)  
- Use UTC timestamps  
- Maintain chronological ordering within files  

Logs failing schema validation are **quarantined** and cannot be used for governance or telemetry aggregation.
