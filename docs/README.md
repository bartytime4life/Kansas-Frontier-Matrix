# docs

## Purpose
Human-facing control plane: doctrine, architecture, ADRs, domain documentation, runbooks, source standards, security, governance, registers, intake, archive, and reports.

## Authority level
canonical

## What belongs here
Markdown documentation, ADRs, runbooks, register READMEs, intake tickets, archived deprecated docs.

## What does not belong here
Executable code, machine schemas, policy code, raw or processed data, release manifests, receipts, or proofs.

## Inputs
See related folders.

## Outputs
See related folders.

## Validation
See `tests/` and `tools/validators/`.

## Review burden
Maintainer review; steward review for trust-bearing changes.

## Related folders
`control_plane/` indexes; `contracts/` defines meaning; `schemas/` defines shape; `policy/` enforces; `tests/` proves.
