## üìò Overview

Telemetry logs record critical metadata about each stage of the cross-domain analytical workflow ‚Äî from data ingestion through model training to results publishing.  
Each log file captures:  
- Analysis and model identifiers  
- Dataset version(s) used  
- Method and script version(s)  
- Execution environment (tool versions, date/time, run id)  
- FAIR+CARE score and audit outcomes  
- Provenance links to results and release manifest  

These logs are essential for auditability, traceability, reproducibility, and governance review across KFM.

## üß© Telemetry Log Convention

### File Naming Scheme  
- `analysisId_yyyy-MM-dd_HHMMSS.json` or `.csv`  
- Example: `crossdomain_carbonwater_2025-11-08T143221.json`

### JSON Structure (snippet example)

```json
{
  "log_id": "CD-CWC-2025-11-08-run01",
  "analysis_id": "crossdomain_carbonwater_v10",
  "datasets": [
    "hydrology_climate_merge.csv: v2025-10-31",
    "eco_hydro_biodiversity.geojson: v2025-10-29",
    "carbon_flux_observations.nc: v2025-10-30"
  ],
  "methods_version": "v10.0.0",
  "model_version": "v10.0.0",
  "toolchain": {
    "python": "3.11.2",
    "xgboost": "1.7.5",
    "shap": "0.41.0"
  },
  "faircare_score": 96.9,
  "run_timestamp": "2025-11-08T14:32:21Z",
  "provenance_link": "../results/model-card-crossdomain_v10.md",
  "status": "success"
}
```

### CSV Structure Example  
```
log_id,analysis_id,dataset,version,usage_count
CD-LGH-2025-11-08-run01,crossdomain_landuse_history_v10,historical_treaty_boundaries.geojson,v2025-11-01,1
...
```
