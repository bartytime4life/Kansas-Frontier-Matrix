## üìò Overview

The telemetry-logs directory captures detailed provenance of the historical analyses workflows, including:
- **Execution logs**: timestamps, dataset versions, script hashes, node/time allocations  
- **Sustainability logs**: compute resource usage, energy estimates, carbon metrics  
- **Audit logs**: FAIR+CARE compliance check results, release linkage, version control hashes  
- **Metadata logs**: checksum records, data lineage, knowledge graph ingestion status  

These logs support reproducibility, governance review, and long-term archiving of historical analysis artefacts.

## ‚öôÔ∏è Log Schema & Standards

Telemetry logs conform to structured formats (JSON, CSV) and follow best-practice schema design:  
- include standardized timestamp (`ISO 8601`), run identifiers, dataset hashes, script versions  
- avoid embedding sensitive data or personally identifiable information in logs. :contentReference[oaicite:0]{index=0}  
- maintain schema consistency across runs to enable automated ingestion and auditing. :contentReference[oaicite:1]{index=1}  

### Example schema fields

| Field             | Description                                | Example                                   |
|-------------------|--------------------------------------------|-------------------------------------------|
| `run_id`          | Unique identifier for the analysis run     | `hist_run_2025_11_08_v10_2_0`             |
| `timestamp_start` | ISO 8601 start time                        | `2025-11-08T21:35:00Z`                     |
| `timestamp_end`   | ISO 8601 end time                          | `2025-11-08T23:45:00Z`                     |
| `script_hash`     | SHA-256 hash of the generating script      | `4f3a...d12e`                              |
| `datasets_used`   | Array of data source IDs                   | `[khs_settlement_1850, usgs_topo_1890]`   |
| `energy_kgCO2eq`  | Estimated carbon equivalent usage          | `5.2`                                     |
| `faircare_status` | Compliance status                          | `PASS`                                    |
| `checksum_manifest`| SHA-256 hash of the manifest file         | `7b2c...af4d`                              |
