## üìò Overview

This directory holds **.json** and **.cpuprofile** outputs generated by the `MapLibre Rendering Performance Playbook`.  
Each file records a snapshot of rendering performance at a specific time, zoom level, and dataset configuration, allowing reproducible benchmarking across KFM deployments.

Profiles are automatically generated by CI/CD or manual testing scripts under:

```bash
docs/guides/geo/snippets/benchmark.ts
```

and stored under structured filenames:

```
<scene>-<zoom>-<timestamp>.json
```

## ‚öôÔ∏è Profiling Metadata Schema

Each `.json` profile follows a lightweight schema aligned with  
`schemas/telemetry/web-render-profiles-v1.json`:

```json
{
  "scene": "kansas_focus",
  "zoom": 12,
  "hardware": "Mac M1 Pro",
  "tileSource": "roads.pmtiles",
  "layers": 42,
  "frameMetrics": {
    "p50_ms": 12.3,
    "p90_ms": 16.5,
    "p99_ms": 23.2
  },
  "memoryMB": 384,
  "cacheSize": 1024,
  "maplibreVersion": "3.6.0",
  "timestamp": "2025-11-09T13:00:00Z"
}
```

> Each record is validated in CI via JSON schema checks; mismatches trigger a `geo-profile-validate` workflow failure.

## üßÆ How to Add a New Profile

1. Run benchmark script:
   ```bash
   npm run benchmark:maplibre
   ```
2. Commit new profile under device/zoom subfolder:
   ```bash
   git add docs/guides/geo/profiles/mobile/android_pixel7_12z.json
   git commit -m "Add mobile render profile for v10.0.0"
   ```
3. CI automatically validates and merges aggregated deltas into `reports/summary-table.json`.
