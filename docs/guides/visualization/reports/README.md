# üìë **Kansas Frontier Matrix ‚Äî Visualization Reports & Accessibility Audit Summaries**  
`docs/guides/visualization/reports/README.md`

**Purpose**  
Serve as the **central reporting hub** for all **visualization validation reports**,  
**accessibility audits**, and **FAIR+CARE v2 compliance summaries** in the Kansas Frontier Matrix (KFM).  

This directory aggregates **WCAG 2.1 AA**, **FAIR+CARE v2**, **Telemetry v2**, and **governance** outputs  
for MapLibre UIs, timeline views, explainability dashboards, and other visualization surfaces.

# üìò Overview

The **Visualization Reports** directory contains validated outputs from:

- UI accessibility audits (Lighthouse, axe, WCAG 2.1 AA)  
- FAIR+CARE v2 ethical visual checks (sensitive content, cultural sites, narratives)  
- Performance and sustainability telemetry (FPS, latency, energy, CO‚ÇÇ)  
- Timeline & explainability visualization validations  
- Governance ledger sync summaries related to visualization components  

Reports here are:

- generated by CI/CD workflows  
- version-controlled  
- referenced by Governance Ledger & dashboards  
- used by FAIR+CARE Council and maintainers during reviews

# üß© Unified Visualization Report Schema (v2)

All reports SHOULD follow this structure (superset):

| Field             | Description                                       | Example                                        |
|-------------------|---------------------------------------------------|------------------------------------------------|
| `report_id`       | Unique workflow/audit identifier                  | `"ui-audit-2025-11-16-0012"`                   |
| `component`       | Visualization component or subsystem              | `"Timeline Visualization"`                     |
| `audit_type`      | Audit type (`"Accessibility"`, `"FAIR+CARE"`, `"Performance"`) | `"Accessibility"`                   |
| `pipeline`        | Source workflow or pipeline                       | `"ui-accessibility-validate.yml"`              |
| `status`          | `"success"|"failure"|"noop"`                       | `"success"`                                    |
| `metrics`         | Object of key metrics                             | `{ "wcag_level": "AA", "fps": 59.2, ... }`     |
| `faircare_status` | `"pass"|"fail"`                                   | `"pass"`                                       |
| `iso_alignment`   | Related ISO standards (if any)                    | `["ISO 9241-210"]`                             |
| `telemetryRef`    | Path/URI to Telemetry v2 NDJSON                   | `"data/telemetry/web-ui.ndjson"`               |
| `ledgerRef`       | Path/URI to ledger entries                        | `"docs/reports/audit/data_provenance_ledger.jsonl"` |
| `timestamp`       | ISO 8601 UTC timestamp                            | `"2025-11-16T12:00:00Z"`                       |

# üß≠ Developer Usage

When you add or modify visualization UIs:

1. Ensure relevant CI workflows produce updated reports under this directory.
2. Confirm reports follow the **Unified Visualization Report Schema (v2)**.
3. Update any Governance Ledger references when new reports are added.
4. Run A11y tests locally where possible (Lighthouse/axe) and check JSON output.
5. Ensure reports include enough context for FAIR+CARE Council review, without exposing PII or sensitive content.
