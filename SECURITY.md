# Security policy

KFM enforces deny-by-default exposure, least privilege, and auditability
at every public boundary. See `infra/hardening/` and `docs/security/`.

## Reporting vulnerabilities
Email security@kfm.local (placeholder). Do not file public issues for
security-sensitive defects.

## What KFM never publishes
- DNA segments at public visibility
- Living-person genealogy without consent
- Exact rare-species locations
- Exact precontact archaeological coordinates
- Critical-infrastructure interior detail
- RAW / WORK / QUARANTINE artifacts on public surfaces
- Unsigned releases
