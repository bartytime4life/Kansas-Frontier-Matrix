# contracts

## Purpose
Semantic meaning of every KFM object family. Pairs 1:1 with `schemas/`. Contracts say *what objects mean*; schemas say *what they look like*.

## Authority level
canonical

## What belongs here
Markdown contract documents per object family.

## What does not belong here
Executable validation, JSON Schema, policy code, source data.
