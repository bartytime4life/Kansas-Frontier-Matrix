## Placement note

`schemas/` is the machine-checkable shape root. New schema definitions should follow the accepted schema-home rule and should not be added here until stewards confirm that this is the correct home.

Cross-domain notes should point back to accepted owning schema lanes when those lanes are verified.

## What belongs here

- README notes.
- Links to child index lanes.
- Links to accepted schema homes once confirmed.
- Migration notes.
- Drift notes.

## What does not belong here

- Canonical schema definitions before placement is confirmed.
- Duplicate schema files.
- Contract prose.
- Validator code.
- Data payloads.
