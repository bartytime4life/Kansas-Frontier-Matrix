# release/source_role_anti_collapse

Draft release review lane for source role checks.

## Purpose

This directory is for release review notes that confirm source role information remains visible and traceable during release preparation.

## Boundary

Keep review notes and links here.

Do not store data payloads, source descriptors, schemas, contracts, policy files, validator code, or published artifacts in this lane.

## Review pointers

A review note should link the relevant governed records:

- release target
- evidence record, when applicable
- validation receipt, when applicable
- manifest, when applicable
- decision record, when applicable
- correction record, when applicable
- changelog entry, when applicable
