## üìö Purpose

The `src/` directory is the **engine core** of the **Kansas Frontier Matrix (KFM)** ‚Äî where raw, unstructured history, geography, and ecology transform into **structured, queryable knowledge**.

It contains deterministic, idempotent, and auditable **pipelines**, **AI/ML enrichments**, and **graph-driven APIs** that power every layer of the system.

| Domain | Function |
| :-- | :-- |
| ‚öôÔ∏è **ETL Pipelines** | Fetch, transform, and normalize spatial and textual sources into FAIR-compliant datasets |
| ü§ñ **AI/ML Modules** | NLP, entity linking, summarization, classification, and confidence reasoning |
| üï∏ **Knowledge Graph** | Neo4j schema using CIDOC CRM, OWL-Time, PROV-O for semantic lineage |
| üîå **API Services** | FastAPI/GraphQL endpoints with field-selective, ETag-cached queries |
| üìä **Autonomous Validation** | Self-auditing workflows for checksum, provenance, schema, and performance metrics |

## ‚öôÔ∏è Workflow Examples

### üß± Add a New Dataset

1. Create manifest ‚Üí `data/sources/{dataset}.json`
2. Run:

   ```bash
   make fetch process stac
   ```
3. Register graph entities via `src/graph/migrations/`
4. Validate with:

   ```bash
   make stac-validate
   ```
5. Document the lineage in `docs/data-contracts/`

### ‚öôÔ∏è Add a New API Endpoint

1. Create route ‚Üí `src/api/routes/{endpoint}.py`
2. Add model schemas ‚Üí `src/api/models/{schema}.py`
3. Register route in `src/api/main.py`
4. Add test case ‚Üí `tests/api/test_{endpoint}.py`
