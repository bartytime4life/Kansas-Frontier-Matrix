## ğŸ“¦ Telemetry Record Structure

Each telemetry file contains **one JSON object per line**, representing events for that run.

### Required Fields

| Field | Description |
|-------|-------------|
| `timestamp` | UTC timestamp of run |
| `run_id` | GitHub Actions run or UUID |
| `incoming_count` | Items polled from STAC API |
| `etag_used` | ETag string or null |
| `items_validated` | Count of items passed |
| `items_quarantined` | Count of items failed |
| `schema_errors` | Count of schema failures |
| `care_errors` | CARE rule violations |
| `publish_count` | Items successfully published |
| `publish_latency_ms` | Write + graph hydration latency |
| `energy_wh` | Estimated energy |
| `co2_g` | Estimated COâ‚‚ output |
| `error_classes` | Array of error categories |
| `ingest_version` | Orchestrator version |
| `governance_status` | pass / fail / escalated |

### Optional Fields

- `quarantine_path`
- `collection_id`
- `dataset_ids`
- `masking_applied`
- `graph_latency_ms`
- `lineage_checksum`
