## Purpose

This lane exists to prove the Atmosphere knowledge-character anti-collapse rule.

Its durable question is:

> Given an Atmosphere object or claim, can the system preserve what epistemic kind it is across validation, evidence resolution, policy, lifecycle state, release, outward presentation, correction, and rollback—without upgrading or flattening its authority?

A mature suite should establish all of the following:

1. each governed Atmosphere object carries one accepted knowledge character when required;
2. missing, unknown, contradictory, or multiply assigned character state fails closed;
3. knowledge character remains distinct from source role, object family, claim type, measurement unit, time kind, and release state;
4. AQI/report context is never interpreted as pollutant concentration;
5. AOD or a remote-sensing mask is never interpreted as ground PM2.5;
6. model, forecast, reanalysis, or fusion output is never presented as direct observation;
7. low-cost sensor output cannot support strong public claims without correction, caveats, confidence, limitations, and review appropriate to use;
8. climate aggregate/anomaly context is not presented as a station or per-place event;
9. advisory context never replaces the official issuing authority;
10. network/site context does not become observation truth or uncontrolled exact-location exposure;
11. re-characterization follows new identity and correction lineage rather than in-place mutation;
12. default tests are synthetic, deterministic, no-network, and safe to inspect;
13. outward surfaces preserve the character and caveats or narrow, abstain, deny, hold, or error;
14. test success remains subordinate to evidence, policy, review, release, correction, and rollback.

This lane does not define the vocabulary, schema, registry, policy, validator, evidence, or release authority it tests.

## Knowledge-character vocabulary posture

### What is confirmed

The Atmosphere docs name an umbrella concept plus eleven specific knowledge characters:

| # | Current documented term | Epistemic role | High-risk collapse |
|---:|---|---|---|
| 0 | `Knowledge character` | Umbrella field/concept | Treating character as optional display metadata. |
| 1 | `OBSERVED_SENSOR` | Direct instrument reading | Model, report, aggregate, or proxy presented as measurement. |
| 2 | `PUBLIC_AQI_REPORT` | Agency AQI/report context | AQI presented as concentration. |
| 3 | `REGULATORY_ARCHIVE` | Archived regulatory record/determination | Vintage archive presented as live current value. |
| 4 | `LOW_COST_SENSOR` | Community/consumer-grade sensor reading | Uncorrected or uncaveated output presented as high-authority observation. |
| 5 | `ATMOSPHERIC_MODEL_FIELD` | Model, forecast, reanalysis, or CTM field | Model presented as observation. |
| 6 | `REMOTE_SENSING_MASK` | Satellite raster, mask, AOD, smoke/fire proxy | AOD/mask presented as ground PM2.5. |
| 7 | `CLIMATE_ANOMALY_CONTEXT` | Baseline-relative aggregate/anomaly context | Aggregate presented as station/per-place event. |
| 8 | `DERIVED_FUSION` | Multi-source derivative/fusion product | Fusion output presented as direct observation. |
| 9 | `METEOROLOGICAL_CONTEXT` | Supporting meteorological context | Context variable presented as the primary variable/claim. |
| 10 | `ALERT_AND_ADVISORY_CONTEXT` | Advisory/referral context | KFM presented as official alert or life-safety authority. |
| 11 | `NETWORK_AND_SITE_CONTEXT` | Station, network, siting, and equipment metadata | Site context presented as observation truth or exact public disclosure approval. |

### What is not settled

The exact machine enum remains open. In particular:

- whether the umbrella phrase is a field name, a registry concept, or an enum value;
- exact casing and serialization;
- deprecated, alias, and migration behavior;
- whether domain-specific terms implement or extend a cross-domain vocabulary;
- registry identity, versioning, digest, and status fields;
- canonical registry home;
- relation between `LOW_COST_SENSOR` and admission-time source role;
- whether all Atmosphere object families require a single character or some role-dependent projection.

Tests must not locally freeze these questions. Until the enum and registry are accepted, use contract-bound fixture intent and explicit truth labels rather than inventing a parallel enum.

## Knowledge character, source role, object family, and claim type

These axes are related but not interchangeable.

| Axis | Question | Example | Authority-collapse risk |
|---|---|---|---|
| Source role | What authority does the source have at admission? | observed, regulatory, modeled, aggregate, candidate | Upcasting a candidate or modeled source. |
| Knowledge character | What epistemic kind is this Atmosphere object? | `PUBLIC_AQI_REPORT`, `REMOTE_SENSING_MASK` | Treating a report, mask, or model as observation. |
| Object family | What governed object is this? | `AirObservation`, `AODRaster`, `ForecastContext` | Renaming fields to change object meaning. |
| Claim type | What statement is being made? | concentration, trend, advisory context, station metadata | Using compatible data to support an incompatible claim. |
| Unit/averaging basis | What quantity and period are represented? | µg/m³ hourly mean, AQI category | Comparing or substituting non-equivalent quantities. |
| Time kind | When was it observed, valid, issued, retrieved, processed, or released? | observed time vs forecast valid time | Presenting stale or forecast state as current observation. |
| Release state | Is this candidate, reviewed, released, superseded, or withdrawn? | candidate vs published | Treating test/catalog presence as publication. |

Required test principle:

```text
A valid combination is not inferred from one field alone.
```

A record with `source_role=observed` does not automatically become `OBSERVED_SENSOR`. An object named `PM25Observation` does not prove the payload is a ground PM2.5 observation. A layer title does not redefine the object. An AI answer cannot upgrade the knowledge character.

| Missing character | Required object omits character. | `INVALID` or `DENY`. |

## Per-character test contracts

### `OBSERVED_SENSOR`

Minimum proof:

- instrument/platform identity and method are explicit;
- source role and object family support direct observation;
- observed time, units, averaging period, QA/calibration, and evidence are present;
- model, report, archive, aggregate, and proxy inputs cannot use this character by renaming;
- public exposure follows review/release requirements.

### `PUBLIC_AQI_REPORT`

Minimum proof:

- issuer, issue time, category/index method, freshness, and evidence are explicit;
- AQI is not inserted into concentration fields;
- public wording says report/index context, not measured concentration;
- stale report context cannot be presented as current;
- AI corrects or abstains when asked to convert AQI directly into concentration without an accepted method and basis.

### `REGULATORY_ARCHIVE`

Minimum proof:

- agency/program, archive vintage, retrieval time, record validity, and lineage are explicit;
- archive state is not presented as live current state;
- a regulatory archive does not automatically confer current regulatory determination;
- superseded versions remain traceable and non-current.

### `LOW_COST_SENSOR`

Minimum proof:

- device/network class and low-cost status are explicit;
- accepted correction/calibration method and version are named;
- confidence, limitations, caveats, time, and comparison basis are present;
- missing caveat closure restricts or denies strong public claims;
- the record cannot silently become reference-grade observation.

### `ATMOSPHERIC_MODEL_FIELD`

Minimum proof:

- model identity, run/issue time, valid time, grid/resolution, ensemble/scenario, and uncertainty are explicit;
- model value is never asserted as observed;
- comparison with observations preserves both characters;
- outdated runs and superseded products are not silently current;
- AI and map labels include modeled/forecast/reanalysis posture.

### `REMOTE_SENSING_MASK`

Minimum proof:

- platform/product, algorithm, version, retrieval time, resolution, cloud/quality mask, and uncertainty are explicit;
- AOD or smoke mask is not PM2.5 ground concentration;
- remote-sensing detection is not exposure or impact proof;
- pixel/mask existence does not become confirmed ground event without accepted supporting evidence;
- public surfaces expose proxy/remote-sensing posture.

### `CLIMATE_ANOMALY_CONTEXT`

Minimum proof:

- baseline period, reference dataset, aggregation unit, variable, and method are explicit;
- anomaly/normal context is not a station observation or current event;
- scenario, trend, attribution, and observed event claims remain distinct;
- per-place wording is denied when only aggregate context exists.

### `DERIVED_FUSION`

Minimum proof:

- every input retains source role, knowledge character, version, time, evidence, and uncertainty;
- fusion method and output uncertainty are explicit;
- fused result cannot present itself as a direct observation;
- missing or incompatible input lineage fails closed;
- correction of an input triggers deterministic invalidation/recomputation posture.

### `METEOROLOGICAL_CONTEXT`

Minimum proof:

- context variable, source role, method, units, and time are explicit;
- supporting meteorology does not become pollutant, smoke, impact, or exposure truth;
- modeled and observed meteorological context remain distinguishable;
- cross-domain joins preserve ownership and caveats.

### `ALERT_AND_ADVISORY_CONTEXT`

Minimum proof:

- official issuer, issue/effective/expiry times, source link/ref, and advisory status are explicit;
- KFM output identifies itself as context and redirects to the issuing authority;
- no test fixture contains actionable emergency instruction presented as KFM authority;
- expired, superseded, canceled, or ambiguous advisory context fails closed;
- health, exposure, evacuation, and protective-action wording is not invented.

### `NETWORK_AND_SITE_CONTEXT`

Minimum proof:

- network/operator/program, site class, equipment, metadata version, and validity interval are explicit;
- site metadata cannot become observation value;
- exact or security-sensitive site information follows policy/release constraints;
- station/network metadata changes create versioned lineage;
- public summaries avoid reverse-engineerable private or sensitive detail.
